Overview

This project involved setting up Active Directory Domain Services (AD DS) on a Windows Server 2016 VM, designating it as the Domain Controller for the savn.local network, and joining two additional VMs (Server 2022 and Windows 10) to the domain. The configuration also required DNS setup and validation of domain membership.

Key Steps

  • Installed AD DS and DNS Server roles on Windows Server.
  • Promoted the server to a Domain Controller for the savn.local domain.
  • Joined additional Windows Server and Windows 10 VMs to the domain.
  • Verified successful configuration using Active Directory Users and Computers (ADUC).

Skills Learned

  • Windows Server administration and role management
  • Active Directory Domain Services (AD DS) configuration
  • DNS setup and troubleshooting
  • Joining and managing computers in a domain
  • System administration best practices (naming conventions, connectivity checks)
