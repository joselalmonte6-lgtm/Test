Project Two — Theater Inventory System (Hash Tables)

  • Back to Resume

Description

This project is a C program that simulates a theater’s inventory management system using a hash table with separate chaining. The program allows buying supplies, selling items, and updating item prices while maintaining accurate logs of inventory and cash flow. At the end of the simulation, it reports both the total cash on hand and the overall complexity of operations.

Features

  • Buy items from suppliers (tracks quantity and cost).
  • Sell items to customers (handles partial stock situations).
  • Update item sale prices dynamically.
  • Log transactions after each command.
  • Track both total cash balance and command complexity.

Technical Details

  • Implemented in C with a hash table of size 300,007 using separate chaining.
  • Custom hash function: res = (1151 * res + (word[i]-'a')) % size;
  • Used linked lists for collision resolution.
  • Data structures included item, node, and hashtable.
  • Operations run in amortized O(1) time.
